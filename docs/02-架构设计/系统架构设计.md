### 1.2 核心设计理念：本地Python + API大脑

本项目采用**本地Python运行时 + 远程API大脑**的架构模式：

| 组件 | 位置 | 说明 |
|------|------|------|
| **Python运行时** | 本地 | 图像处理、视频编解码、文件IO等计算密集任务 |
| **LLM大脑** | 远程API | 意图理解、任务规划、分析报告生成 |
| **工具箱** | 本地 | OpenCV/FFmpeg等本地库执行具体分析任务 |

**优势**：
- 🖥️ **本地处理**：图像/视频分析在本地执行，保护隐私，无需上传原始数据
- 🧠 **云端智能**：复杂推理和自然语言理解由LLM API提供
- 🔧 **灵活切换**：支持OpenAI/MiniMax/Anthropic多种LLM provider
- 📦 **轻量部署**：只需Python环境，无需GPU本地部署

### 1.3 分层说明

| 层次 | 职责 | 关键组件 |
|------|------|----------|
| 用户接口层 | 接收用户输入、返回结果 | CLI命令行 (chat/image/video) |
| 应用层 | 任务编排、流程控制 | TaskEngine、IntentParser、ResponseBuilder |
| 能力层 | **API调用远程LLM** | LLMClient (MiniMax/OpenAI/Anthropic) |
| 工具层 | 本地图像/视频处理、ISP分析 | ImageTools、VideoTools、ISPAnalysis |
| 基础设施层 | 底层计算、IO | Python 3.9+、OpenCV、FFmpeg、NumPy |

## 4. API设计

### 4.1 LLM API配置（本地Python调用远程服务）

```yaml
# config/config.yaml
llm:
  # 默认使用 MiniMax (国内访问快)
  provider: "minimax"
  model: "MiniMax-M2.5"
  api_key: "${MINIMAX_API_KEY}"
  base_url: "https://api.minimax.chat/v1"
  
  # 可切换到 OpenAI
  # provider: "openai"
  # model: "gpt-4o"
  # api_key: "${OPENAI_API_KEY}"
  # base_url: "https://api.openai.com/v1"
  
  # 可切换到 Anthropic
  # provider: "anthropic"
  # model: "claude-sonnet-4-20250514"
  # api_key: "${ANTHROPIC_API_KEY}"
  # base_url: "https://api.anthropic.com"
```

### 4.2 环境变量配置

```bash
# ~/.bashrc 或 ~/.zshrc

# MiniMax (默认)
export MINIMAX_API_KEY="your-api-key-here"

# 或 OpenAI
# export OPENAI_API_KEY="sk-xxx"

# 或 Anthropic
# export ANTHROPIC_API_KEY="sk-ant-xxx"
```

## 5. 部署设计

### 5.1 本地开发环境（Python 3.9+）

```bash
# 检查Python版本
python3 --version  # 需要 3.9+

# 创建虚拟环境 (推荐)
cd isp-agent
python3 -m venv venv
source venv/bin/activate  # macOS/Linux
# 或
venv\Scripts\activate     # Windows

# 安装依赖
pip install -r requirements.txt
```

### 5.2 API密钥配置

```bash
# 方式1: 环境变量 (推荐)
export MINIMAX_API_KEY="your-key-here"

# 方式2: 配置文件 (注意不要提交到git)
# 编辑 config/config.yaml 填入API密钥
```
