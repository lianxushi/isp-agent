## 一、产品概述

### 1.1 产品定位

**ISP-Agent (OpenClaw Edition)** 是一款基于 **OpenClaw 平台**的 ISP 领域专业智能助手，通过自然语言交互帮助工程师完成图像质量分析、ISP 参数诊断、调参优化建议等任务。

**一句话定位**: 
> 工程师口袋里的 ISP 专家，随时随地解决图像质量问题

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| **降门槛** | 新人无需记忆复杂参数，通过对话即可获得专业建议 |
| **提效率** | 自动化分析 + AI 建议，大幅缩短调参周期 |
| **保经验** | 沉淀专家经验，形成可传承的知识库 |
| **可扩展** | 基于 OpenClaw 插件机制，持续扩展能力 |

### 1.3 技术选型

| 组件 | 技术选择 | 理由 |
|------|----------|------|
| **平台** | OpenClaw | 插件化架构成熟，多通道支持 |
| **LLM** | MiniMax-M2.5 (默认) | 性价比高，支持 Function Call |
| **图像处理** | Python + OpenCV | 成熟稳定，ISP 领域标配 |
| **视频处理** | FFmpeg + OpenCV | 全格式支持 |
| **知识库** | 向量数据库 (Qdrant) | 可选，用于案例检索 |
| **部署** | Docker | 一键部署，开箱即用 |

### 3.2 典型对话示例

#### 场景 1: 图像问题诊断

```
用户: [上传图像] "帮我看看这张图有什么问题"

ISP-Agent:
## 图像质量分析报告

### 基础信息
- 分辨率: 3840×2160 (4K)
- 格式: JPEG

### 质量评分: 72/100
| 维度 | 得分 | 状态 |
|------|------|------|
| 清晰度 | 70 | ⚠️ 轻微偏软 |
| 噪声控制 | 65 | ⚠️ 中等噪声 |
| 色彩准确 | 85 | ✅ 良好 |

### 问题诊断
1. 暗部噪声明显 (估计 ISO 1600)
2. 边缘略微模糊

### 调参建议
- noise_reduction: 50 → 65
- sharpening: 0 → 30
```

#### 场景 2: ISP 参数咨询

```
用户: "AE exposure_bias 取值范围是多少？什么场景用？"

ISP-Agent:
## AE exposure_bias 详解

### 基本概念
曝光补偿，调整自动曝光的测光结果。
正值使图像变亮，负值使图像变暗。

### 取值范围
- 典型范围: -2.0EV ~ +2.0EV
- 部分平台支持: -3.0EV ~ +3.0EV

### 使用场景
| 场景 | 推荐值 | 原因 |
|------|--------|------|
| 逆光主体 | +0.5~+1.5EV | 补偿背景过亮 |
| 雪景/沙滩 | -0.5~-1.0EV | 避免过曝 |
| 夜景人像 | +0.3~+0.7EV | 主体亮度适中 |
```

### 5.2 核心技能 - ISP 调参助手

**文件**: `skills/isp_tuning/SKILL.md`

```markdown
# ISP 调参助手

## 核心能力
1. **参数解释**: 解释 ISP 参数的作用、取值范围
2. **场景推荐**: 根据场景特点推荐参数配置
3. **问题诊断**: 识别参数配置问题并给出修正建议

## 支持的参数模块

### AE (Auto Exposure)
| 参数 | 范围 | 说明 |
|------|------|------|
| exposure_bias | -3.0~+3.0 EV | 曝光补偿 |
| meter_mode | center/spot/matrix | 测光模式 |
| ae_target | 50~80% | 曝光目标亮度 |

### AWB (Auto White Balance)
| 参数 | 范围 | 说明 |
|------|------|------|
| wb_mode | auto/daylight/cloudy/tungsten | 白平衡模式 |
| rg_boost | 0~4095 | R 通道增益 |
| bg_boost | 0~4095 | B 通道增益 |

### Noise Reduction
| 参数 | 范围 | 说明 |
|------|------|------|
| nr_mode | off/temporal/spatial/spatiotemporal | 降噪模式 |
| luma_strength | 0~100 | 亮度降噪 |

### HDR / LDC / LFM
| 参数 | 范围 | 说明 |
|------|------|------|
| HDR_enable | true/false | HDR 开关 |
| ldc_strength | 0~100 | 畸变校正强度 |
| lfm_enable | true/false | LED 闪烁抑制 |

## 工具使用
- 使用 `isp_image_analysis` 分析当前图像质量
- 使用 `isp_video_analysis` 分析视频质量
```

## 八、与 Imatest 的定位差异

### 8.1 功能定位对比

| 维度 | Imatest | ISP-Agent | 关系 |
|------|---------|-----------|------|
| **核心定位** | 精确测量仪器 | 智能助手 | 互补 |
| **使用场景** | 实验室、认证测试 | 现场、快速诊断、日常 | 互补 |
| **输出形式** | 精确数值、报告 | 自然语言建议 | 不同 |
| **学习门槛** | 高（需专业培训） | 低（自然语言） | ISP-Agent 更易用 |
| **测量精度** | 微米级 | 趋势级 | Imatest 更精确 |

### 8.2 Imatest 核心功能参考

| Imatest 功能 | 说明 | ISP-Agent 对应 |
|--------------|------|---------------|
| SFR (Spatial Frequency Response) | MTF 曲线测量 | 清晰度趋势分析 |
| Colorcheck | 色卡色彩准确性 | 色彩分析 |
| Noise | 噪声测量 (SNR) | 噪声分析 |
| Dynamic Range | 动态范围 | 动态范围分析 |
| Blemish | 坏点/暗点检测 | 坏点检测 |

> 具体指标（SNR、MTF 阈值等）由石先生在开发完成后提供。
