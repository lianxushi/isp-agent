# ISP-Agent 工具接口设计

## 1. 工具总览

| 工具名称 | 描述 | 优先级 | 状态 |
|----------|------|--------|------|
| `isp_image_analysis` | 图像质量分析 | P0 | 📋 |
| `isp_video_analysis` | 视频质量分析 | P1 | 📋 |
| `isp_diagnosis` | ISP 问题诊断 | P1 | 📋 |
| `isp_config_parser` | ISP 配置解析 | P0 | 📋 |
| `isp_report` | 生成分析报告 | P1 | 📋 |

## 2. isp_image_analysis

### 2.1 工具定义

```typescript
const ispImageAnalysisTool = {
  name: "isp_image_analysis",
  description: `分析图像的 ISP 质量指标。

使用场景：
- 用户上传图像要求分析质量
- 用户询问图像有什么问题
- 用户需要生成图像质量报告
- 用户上传视频抽帧后分析单帧质量

返回内容：
- 基础信息 (分辨率、格式、大小)
- EXIF/DNG 元数据 (相机、镜头、ISP 参数)
- 直方图统计 (RGB/L 通道)
- 噪声分析 (亮度噪声、色度噪声、噪声等级)
- 色彩分析 (白平衡、饱和度、色调)
- 动态范围 (高光/暗部细节)
- 清晰度 (边缘锐度、纹理)
- 综合评分 (0-100)`,
  
  inputSchema: {
    type: "object",
    properties: {
      image_path: {
        type: "string",
        description: "图像文件路径 (支持 JPEG/PNG/TIFF/DNG/RAW)"
      },
      analysis_type: {
        type: "string",
        enum: ["full", "quick", "noise", "color", "sharpness", "dynamic"],
        default: "full",
        description: "分析类型: full=完整分析, quick=快速分析"
      },
      output_format: {
        type: "string",
        enum: ["json", "markdown"],
        default: "markdown",
        description: "输出格式"
      }
    },
    required: ["image_path"]
  }
}
```

### 2.2 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `image_path` | string | ✅ | 图像文件路径 |
| `analysis_type` | enum | ❌ | 分析类型，默认为 full |
| `output_format` | enum | ❌ | 输出格式，默认为 markdown |

### 4.2 输出格式

```markdown
## ISP 问题诊断报告

**问题描述**: 图像整体偏暗，夜景噪声严重

---

### 🔍 可能原因分析

#### 原因 1: AE 曝光不足
**概率**: 85%
**说明**: 当前场景光照不足，AE 未正确曝光
**检测方法**: 查看 EXIF 的 ISO/快门/光圈组合

**推荐调整**:
| 参数 | 当前值 | 建议值 |
|------|--------|--------|
| ae_target | 50 | 58 |
| exposure_time_max | 16.67ms | 33.33ms |
| gain_max | 36dB | 48dB |

#### 原因 2: 降噪参数过高
**概率**: 45%
**说明**: 强降噪会抹平细节，造成涂抹感

**推荐调整**:
| 参数 | 当前值 | 建议值 |
|------|--------|--------|
| nr_mode | spatiotemporal | temporal |
| temporal_strength | 80 | 50 |

#### 原因 3: Gamma 曲线偏暗
**概率**: 30%
**说明**: Gamma 值过高导致整体偏暗

**推荐调整**:
| 参数 | 当前值 | 建议值 |
|------|--------|--------|
| gamma | 2.4 | 2.2 |

---

### ✅ 建议处理流程

1. **第一步**: 调整 ae_target 到 58，观察亮度改善
2. **第二步**: 如果噪声严重，调整降噪模式为 temporal
3. **第三步**: 如果仍偏暗，微调 gamma 到 2.2

### 📚 相关知识

- [AE 曝光原理](link)
- [降噪模式对比](link)
- [Gamma 曲线调整技巧](link)
```

## 6. Python SDK 设计

### 6.1 Python SDK 结构

```python
# isp_sdk/
# ├── __init__.py
# ├── analyzer.py      # 图像分析
# ├── video.py         # 视频处理
# ├── parser.py        # 配置解析
# ├── metrics.py       # 指标计算
# └── utils.py         # 工具函数

# 主入口
from isp_sdk import ImageAnalyzer, VideoAnalyzer, ConfigParser

# 图像分析
analyzer = ImageAnalyzer()
result = analyzer.analyze("test.jpg", analysis_type="full")
print(result.to_markdown())

# 视频分析  
video = VideoAnalyzer()
result = video.analyze("test.mp4", extract_frames=5)
print(result.to_markdown())

# 配置解析
parser = ConfigParser()
config = parser.parse("isp_config.json")
print(config.to_markdown())
```

### 6.2 核心类定义

```python
from dataclasses import dataclass
from typing import Optional, List
from enum import Enum

class AnalysisType(Enum):
    FULL = "full"
    QUICK = "quick"
    NOISE = "noise"
    COLOR = "color"
    SHARPNESS = "sharpness"
    DYNAMIC = "dynamic"

@dataclass
class NoiseMetrics:
    luma_noise_db: float          # 亮度噪声 (dB)
    chroma_noise_db: float        # 色度噪声 (dB)
    noise_level: str              # low/medium/high
    estimated_iso: int            # 估计 ISO
    hot_pixels: int              # 坏点数
    dead_pixels: int             # 死像素点

@dataclass
class ColorMetrics:
    white_balance_error: float   # 白平衡误差 (K)
    saturation: float            # 饱和度偏离度
    hue_shift: float            # 色调偏移
    color_accuracy_score: float  # 色彩准确度评分

@dataclass
class QualityScores:
    overall: int                 # 综合评分 0-100
    noise: int                   # 噪声评分
    color: int                   # 色彩评分
    sharpness: int               # 清晰度评分
    dynamic_range: int           # 动态范围评分

@dataclass
class AnalysisResult:
    report_id: str
    basic_info: dict
    camera_info: dict
    isp_params: dict
    histogram: dict
    noise_metrics: Optional[NoiseMetrics]
    color_metrics: Optional[ColorMetrics]
    scores: QualityScores
    issues: List[dict]
    suggestions: List[str]
    timestamp: str
    
    def to_markdown(self) -> str:
        """转换为 Markdown 格式"""
        pass
    
    def to_json(self) -> dict:
        """转换为 JSON 格式"""
        pass

class ImageAnalyzer:

        self.config = config or {}

    ) -> AnalysisResult:

        分析图像
        
        Args:
            image_path: 图像路径
            analysis_type: 分析类型
            output_format: 输出格式
            
        Returns:
            AnalysisResult 对象
```

## 7. 工具执行流程

```
用户请求
    │
    ▼
┌─────────────────┐
│ 参数验证        │ ← 检查 image_path 是否存在
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 格式检测        │ ← 检测文件格式
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Python 脚本调用 │ ← spawn Python 进程
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ OpenCV/NumPy    │ ← 执行图像分析
│ 图像处理        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 结果解析        │ ← JSON → Python 对象
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 格式化输出      │ ← Markdown / JSON
└─────────────────┘
```

## 8. 错误处理

| 错误类型 | 处理方式 | 示例 |
|----------|----------|------|
| 文件不存在 | 返回错误信息 | `{"ok": false, "error": "File not found"}` |
| 不支持的格式 | 返回错误信息 | `{"ok": false, "error": "Unsupported format"}` |
| 分析失败 | 返回错误信息 + 日志 | `{"ok": false, "error": "Analysis failed"}` |
| 超时 | 返回部分结果 + 警告 | `{"ok": true, "partial": true}` |
